Task1

人工智能相关背景与有关技术的了解。（机器学习 machine learning、模式识别 pattern recognition、数据挖掘 data mining、计算机视觉 computer vision、自然语言处理 Natural
language processing 等）

  • 机器学习（Machine Learning）

    • 机器学习是人工智能的一个分支。机器学习算法是一类从数据中自动分析获得规律，并利用规律对未知数据进行预测的算法。（wikipedia）

    • 有 监督学习（如分类、回归）、无监督学习（如聚类）、强化学习 几个类别
    • 数据挖掘本质上属于机器学习的内容。（wikipedia）
    • 去年没抽中机器学习的课
  • 模式识别（Pattern Recognition）

    • 模式识别，就是通过计算机用数学技术方法来研究模式的自动处理和判读。从数据（如文本、图像、声音）中自动识别出规律性、结构或特定模式。

    • 对数据进行分类（将数据划分到已知类别）与聚类（无标签数据的自然分组）
    • 原理是基于概率分布与统计学模型
    • 使用 KNN 算法和 SVM
    • 适用于 处理具有层次结构的数据
  • 数据挖掘（data mining）

    • 数据挖掘是一个跨学科的计算机科学分支 。它是用人工智能、机器学习、统计学和数据库的交叉方法在相对较大型的数据集中发现模式的计算过程。（wikipedia）

    • 对大规模数据进行自动或半自动的分析，以提取过去未知的有价值的潜在信息，例如数据的分组（通过聚类分析）、数据的异常记录（通过异常检测）和数据之间的关系（通过关联式规则挖掘）。

    • 涉及六类常见的任务：
      • 异常检测（异常/变化/偏差检测）– 识别不寻常的数据记录，错误数据需要进一步调查。
      • 关联规则学习（依赖建模）– 搜索变量之间的关系。例如，一个超市可能会收集顾客购买习惯的数据。运用关联规则学习，超市可以确定哪些产品经常一起买，并利用这些信息帮助营销。这有时被称为市场购物篮分析。
      • 聚类 – 是在未知数据的结构下，发现数据的类别与结构。
      • 分类 – 是对新的数据推广已知的结构的任务。例如，一个电子邮件程序可能试图将一个电子邮件分类为“正常邮件”或“垃圾邮件”。
      • 回归 – 试图找到能够以最小误差对该数据建模的函数。
      • 汇总 – 提供了一个更紧凑的数据集表示，包括生成可视化和报表。
  • 计算机视觉（computer vision）

    • 是指用摄影机和计算机代替人眼对目标进行识别、跟踪和测量等机器视觉，并进一步做图像处理，用计算机处理成为更适合人眼观察或传送给仪器检测的图像（wikipedia）

    • 核心目标：
      • 感知：识别图像中的物体、场景、动作。
      • 理解：推断图像中的关系与语义。
      • 生成：根据需求创建图像或视频。
    • 这学期选了计算机视觉课，感觉相当硬
  • 自然语言处理（Natural language processing）

    • 自然语言处理是人工智能和语言学领域的分支学科。此领域探讨如何处理及运用自然语言；自然语言处理包括多方面和步骤，基本有认知、理解、生成等部分。（wikipedia）

    • 其发展经历了从早期基于规则和统计的方法到现代深度学习的变革，尤其是Transformer架构（如BERT、GPT）的突破，使得机器能够更精准地捕捉上下文语义并生成连贯文本。

task4

了解最近三年各种有关学术竞赛. 包括 KDD、CVPR、CIKM、ICDM、AAAI、IJCAI、RecSys 等国际会议以及国内如 PRCV、CCDM 等会议这几年举办的与数据分析与挖掘有关的学术类算法竞赛。

  • KDD 2023：聚焦多模态数据挖掘与可解释性AI，竞赛主题包括大规模社交网络异常检测和城市交通流量预测。

  • ICDM 2023：竞赛主题涵盖时序数据异常检测与联邦学习优化。

  • AAAI 2024：其竞赛多关注小样本学习与自动化机器学习（AutoML）。

  • IJCAI 2023–2024：竞赛方向包括多智能体博弈决策与知识图谱补全。

  • CVPR 2024：竞赛聚焦视频理解与3D重建。

  • RecSys 2023：推荐系统竞赛侧重冷启动问题。

  • PRCV 2023：竞赛任务包括遥感图像语义分割与工业质检缺陷检测。

  • CCDM 2024：重点方向为医疗数据挖掘与教育数据分析。

task5

通过网络搜索“数据挖掘竞赛”、”数据挖掘比赛”、“人工智能竞赛”、“大数据竞赛”等了解相关信息。

  • 数据挖掘竞赛（Data Mining Competition）是一种以解决实际问题为导向的竞赛活动，参赛者通过分析给定的数据集，利用机器学习、统计学和数据分析技术构建模型，完成预测、分类、聚类等任务。这类竞赛通常由企业、学术机构或平台组织，旨在推动技术创新、挖掘数据价值，并为参赛者提供实践和展示技能的机会。

  • task2 的 kaggle 和 task3 的天池平台就是知名线上数据挖掘竞赛平台
  • 在知乎上看到了许多关于数据挖掘竞赛的介绍和入门教程

task7

最近以 Chatgpt、Deepseek 为代表的大模型技术成为人工智能发展的重要引擎。请调研国内外相关公司的大模型产品设计的中的相关技术和概念，如 Transformer、Token、MoE、预训练、微调、蒸馏等，以及大模型在众多行业领域的应用场景。 调研的大模型可不局限于国外的如谷歌的 Gemini、Meta 公司的 LLAMA、 OpenAI 的 ChatGPT 系列、XAI 公司的 Grok、Anthropic 公司的 Claude 等以及国内的阿里 Qwen2.5、字节豆包 1.5Pro、腾讯混元、百度文心 4.0、讯飞星火、月之暗面 Kimi、智谱 GLM-4、昆仑万维天工 4.0、百川智能 Baichuan 系列、MiniMax01 系列等。

以deepseek为例

  • 基础架构：Transformer 的深度优化
    • 采用多层 Transformer 结构，通过自注意力机制（Self-Attention）捕捉长距离语义依赖。
    • 稀疏注意力：引入局部敏感哈希（LSH）优化注意力计算，降低复杂度。
    • 动态位置编码：结合旋转位置编码（RoPE）与可学习位置参数，提升模型对序列位置的适应性。
  • Token 化与数据工程
    • 分词策略：采用 Byte-Pair Encoding (BPE) 算法，结合中文单字切分与常用词合并，平衡语义粒度与计算效率。
    • 通过 动态词表扩展技术，在微调阶段自动识别新领域术语。
  • 预训练
    • 数据规模：使用超万亿 Token 的中英混合语料，涵盖网页、书籍、代码等。
    • 训练目标：基于掩码语言建模（MLM）和因果语言建模（CLM）的混合目标。
  • 微调
    • 采用 指令微调（Instruction Tuning） 对齐用户意图，如使用人工标注的对话数据。
    • 引入 参数高效微调（PEFT） 技术（如 LoRA），仅调整部分网络权重。
  • 蒸馏
    • 将千亿级大模型 知识蒸馏 至百亿级模型，保持性能的同时降低推理成本（如 DeepSeek-MoE 系列）。
