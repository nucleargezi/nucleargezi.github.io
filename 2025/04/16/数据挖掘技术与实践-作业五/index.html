测试集比例对分类性能的影响分析

实验结果对比

测试集比例 准确度 平均精度 召回率 F1值
0.5 0.8187 0.7224 0.8030 0.7969
0.3 0.8221 0.7164 0.8333 0.8021
0.1 0.8284 0.7075 0.8545 0.8034

性能变化

  1. 准确度（Accuracy）：

    • 在测试集比例为0.5时最低（0.8187），0.3时略有上升（0.8221），0.1时再次上升（0.8284）。
    • 整体变化幅度不大，说明模型对数据划分比例的变化具有一定的鲁棒性。

  2. 平均精度（Average Precision）：

    • 随着测试集比例增加（训练数据减少）而明显下降：
      • 0.5 → 0.7224
      • 0.3 → 0.7164
      • 0.1 → 0.7075
    • 表明训练数据量减少会影响模型对正类的识别精度。

  3. 召回率（Recall）：

    • 随着测试集比例增加（训练数据减少）而上升：
      • 0.5 → 0.8030
      • 0.3 → 0.8333
      • 0.1 → 0.8545
    • 可能原因：训练数据减少时，模型倾向于更”激进”地预测正类，减少了假阴性。

  4. F1值：

    • 保持相对稳定（0.79-0.81之间），说明精确率和召回率的此消彼长达到了较好的平衡。

现象解释

  1. 训练数据量影响：

    • 当测试集比例从0.5增加到0.9（训练数据从50%减少到10%）时：
      • 平均精度持续下降，表明模型学习能力因数据减少而减弱。
      • 召回率反而上升，可能是模型在数据不足时倾向于”宁可错杀不可放过”的策略。

  2. 指标间的trade-off：

    • 平均精度和召回率呈现相反的变动趋势，这是分类模型中常见的精确率-召回率权衡现象。
    • F1值的稳定性说明模型在不同数据划分下保持了较好的综合性能。

结果分析

  1. 训练集比例的影响：

    • 随着训练集比例降低，所有模型的性能均下降，因为训练数据减少导致模型学习不充分。
    • XGBoost在所有比例下表现最优，其次是GBDT和随机森林。

  2. 模型对比：

    • XGBoost由于引入了正则化和二阶优化，泛化能力最强。
    • GBDT和随机森林性能相近，但随机森林对训练数据的变化更敏感。
